# <a name="determine-hybrid-identity-lifecycle-adoption-strategy"></a>ハイブリッド ID ライフサイクルの導入戦略の決定
このタスクでは、「 [ハイブリッド ID 管理タスクの決定](active-directory-hybrid-identity-design-considerations-hybrid-id-management-tasks.md)」で定義したビジネス要件を満たすために、ハイブリッド ID ソリューションの ID 管理戦略を定義します。

この手順で前述したエンド ツー エンドの ID ライフサイクルに従ってハイブリッド ID 管理タスクを定義するには、ライフサイクルの各フェーズで使用できるオプションを考慮する必要があります。

## <a name="access-management-and-provisioning"></a>アクセス管理とプロビジョニング
優れたアカウント アクセス管理ソリューションを使用すると、組織全体でどのユーザーがどの情報にアクセスできるかを正確に追跡できます。

アクセス制御は、一元化されたプロビジョニング システムの重要な機能です。 機密情報を保護できるだけでなく、既存のアカウントのうち、権限が承認されていないものや不要になったものを、アクセス制御を通じて明確化できます。 不要になったアカウントを制御するために、プロビジョニング システムにより、アカウントを所有するユーザーの権限情報とアカウント情報が関連付けられます。 通常、権限のあるユーザーの ID 情報は、人事部門のデータベースやディレクトリに保管されます。

先進的な IT 企業のアカウントには、権限を定義する何百ものパラメーターが含まれていて、それらの詳細をプロビジョニング システムで制御できます。 権限のあるソースから提供されるデータを使用して新しいユーザーを識別できます。 新しいユーザーに対するリソースのプロビジョニングを承認 (または拒否) するプロセスが、アクセス要求の承認機能によって実行されます。

## <a name="role-based-access-control"></a>ロールベースのアクセス制御
ロールベースのアクセス制御 (RBAC) では、ロールとプロビジョニングのポリシーを使用して、ビジネス プロセスとルールを評価してテストした上で適用し、ユーザーにアクセスを許可します。 キーの管理者は、プロビジョニングのポリシーを作成し、ユーザーにロールを割り当てて、これらのロールのリソースに対する権限のセットを定義します。 RBAC では、ID 管理ソリューションを拡張してソフトウェアベースのプロセスを使用することにより、プロビジョニング中のユーザーによる手動操作を削減します。
Azure AD の RBAC により、ユーザーが Azure portal にアクセスした時点で実行できる操作数を企業が制限できます。 RBAC を使用してポータルへのアクセスを制御することで、IT 管理者は、次のアクセスの管理方法を使用してアクセスを委任できます。

* **グループベースのロールの割り当て**: ローカルの Active Directory から同期できる Azure AD グループにアクセスを割り当てることができます。 これにより、グループ管理用のツールとプロセスに組織が費やした既存の投資を活用できます。 また、Azure AD Premium の委任されたグループ管理機能も使用できます。
* **Azure の組み込みロールの活用**: 所有者、共同作業者、閲覧者の 3 つのロールを使用して、ジョブの実行に必要なタスクのみを実行する権限をユーザーとグループに付与できます。
* **リソースへのきめ細かなアクセス**: 特定のサブスクリプションのユーザーとグループ、リソース グループ、Web サイトやデータベースなどの個々の Azure リソースにロールを割り当てることができます。 このように、ユーザーが必要なすべてのリソースにアクセスし、管理する必要のないリソースには一切アクセスできないようにすることができます。

## <a name="provisioning-and-other-customization-options"></a>プロビジョニングとその他のカスタマイズ オプション
チームは、ビジネスの計画や要件を使用して、ID ソリューションをどの程度カスタマイズするかを決定できます。 たとえば、大企業では、複数の地域にまたがって使用されるアプリケーションを順次プロビジョニングするスケジュールに合わせて、ワークフローやカスタム アダプターの段階的な展開計画を用意する必要が生じる場合があります。 別のカスタマイズ計画により、テストの完了後に、組織全体で 2 つ以上のアプリケーションをプロビジョニングする場合もあります。 ユーザーとアプリケーションのやり取りはカスタマイズでき、自動プロビジョニングに対応するように、リソースのプロビジョニング手順を変更することもできます。

プロビジョニングを解除すれば、サービスまたはコンポーネントを削除できます。 たとえば、アカウントのプロビジョニングを解除すると、アカウントがリソースから削除されます。

リソースのプロビジョニングのハイブリッド モデルでは、要求とロールベースのアプローチを結合します。これらはどちらも Azure AD によってサポートされています。 企業は、従業員または管理対象システムのサブセットに対して、ロールベースの割り当てを使用してアクセスを自動化できます。 また、要求ベースのモデルを使用して、その他すべてのアクセス要求または例外を処理できます。 企業によっては、まず手動で割り当てを行い、徐々にハイブリッド モデルへと発展させて、将来的に完全なロールベースのデプロイを目指すことができます。

また、完全なロールベースのプロビジョニングの実現がビジネス上の理由から現実的でないと判断し、当面の目標としてハイブリッド アプローチを目指すこともできます。 さらに、企業によっては、要求ベースのプロビジョニングのみで問題がないため、ロールベースの自動化されたプロビジョニング ポリシーの定義や管理の追加作業に投資する必要はないと判断する場合もあります。

## <a name="license-management"></a>ライセンスの管理
Azure AD のグループベースのライセンス管理では、管理者がユーザーをセキュリティ グループに割り当てると、Azure AD によって、グループのすべてのメンバーに自動的にライセンスが割り当てられます。 その後でユーザーがグループに追加されるか、グループから削除されると、それぞれに応じて、ライセンスが自動的に割り当てられるか削除されます。

オンプレミスの AD から同期するグループ、または Azure AD で管理するグループを使用できます。 Azure AD Premium のセルフ サービスのグループ管理をこれと組み合わせることにより、ライセンスの割り当てを適切な意思決定者に簡単に委任できます。 ライセンスの競合や位置データの欠落などの問題を自動的に解決できます。

## <a name="self-regulating-user-administration"></a>自動調整ユーザー管理
すべての内部組織に対してリソースのプロビジョニングを開始する際に、自動調整ユーザー管理機能を実装します。 組織の境界を越えてユーザー プロビジョニングの長所とメリットを実現できます。 この環境では、ユーザーの状態に対する変更が、組織の枠や地域を越えて自動的にアクセス権に反映されます。 プロビジョニングのコストを削減し、アクセスと承認のプロセスを効率化できます。 この実装により、ロールベースのアクセス制御の実装が持つ可能性をすべて引き出し、組織でエンド ツー エンドのアクセス管理を実現できます。 ユーザー プロビジョニングの制御手順が自動化されるため、管理コストを削減できます。 セキュリティ ポリシーの適用を自動化することによってセキュリティを強化し、多数のユーザーに対するユーザー ライフサイクル管理とリソース プロビジョニングの効率化と一元化を図ることができます。

## <a name="define-synchronization-management"></a>同期管理の定義
オンプレミスのディレクトリと Azure AD を統合すると、クラウドとオンプレミス両方のリソースにアクセスするための共通の ID が提供されるため、ユーザーの生産性が向上します。 この統合によって、ユーザーや組織には次の利点があります。

* 組織は、Windows Server Active Directory を利用した後に Azure Active Directory に接続して、オンプレミスとクラウドベースのサービス全体で共通するハイブリッド ID をユーザーに提供できます。
* 管理者は、アプリケーション リソース、デバイスおよびユーザーの ID、ネットワークの場所、多要素認証に基づき、条件付きアクセスを提供できます。
* ユーザーは、共通 ID を Azure AD、Office 365、Intune、SaaS アプリ、サードパーティ アプリケーションのアカウントで利用できます。
* 開発者は、共通 ID モデルを利用したアプリケーションを構築し、アプリケーションをオンプレミスの Active Directory またはクラウドベース アプリケーション向け Azure に統合できます。
