# <a name="prebuilt-domain-reference"></a>事前構築済みのドメインのリファレンス
このリファレンスは、[事前構築済みのドメイン](luis-how-to-use-prebuilt-domains.md)に関する情報を提供します。事前構築済みのドメインは、LUIS が提供している意図とエンティティの事前構築済みのコレクションです。

対照的に、[カスタム ドメイン](luis-how-to-start-new-app.md)は意図とモデルがない状態から始まります。 任意の事前構築済みのドメインの意図とエンティティをカスタム モデルに追加できます。

## <a name="calendar"></a>Calendar 

Calendar ドメインには、カレンダー エントリに関する意図とエンティティが用意されています。 Calendar の意図には、予定の追加、削除、または編集、空き状況の確認、カレンダー エントリまたは予定に関する情報の検索などがあります。

### <a name="intents"></a>意図
| 意図の名前 | 説明 | 例 |
| ---------------- |-----------------------|----|
| Add | 新しい 1 回限りの項目をカレンダーに追加します。| 日曜日午後 2 時に Lisa と約束 <br/><br/>会議のスケジュールを設定したい<br/><br/>会議を設定する必要がある|
| CheckAvailability | ユーザーのカレンダーや他のユーザーのカレンダーで、予定や会議の空き状況を検索します。| Jim はいつ会うことができる? <br/><br/>明日、Carol が空いている時間を表示して<br/><br/>Chris は土曜日に空いている?|
| Delete | カレンダー エントリを削除する要求。| Carol との約束をキャンセルして。 <br/><br/>午前 9 時の会議を削除して<br/>|
| Edit | 既存の会議またはカレンダー エントリを変更する要求。| 午前 9 時の会議を午前 10 時に変更して。<br/><br/>自分のスケジュールを更新したい。<br/><br/>Ryan との会議予定を変更して。|
| Find | 自分の週間カレンダーを表示します。| 歯科の診察予定を探して。 <br/><br/>カレンダーを表示して<br/>|

### <a name="entities"></a>エンティティ
| エンティティ名 | 説明 | 例 |
| ---------------- |-----------------------|----|
| Location | カレンダー項目、会議、予定の場所。 たとえば、住所、都市、地域などです。| 209 ナッシュビル ジム <br/><br/>897 パンケーキ ハウス<br/><br/>ガレージ|
